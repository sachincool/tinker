Anatomy · silicon → pod
What a GPU pod actually sits on
Twelve layers. Each one trusts the layer below and misleads the layer above when it breaks.
Layer & its one jobWhere it breaks
12
workload · pod
the training job or inference server you actually wanted
training is all-or-nothing on N GPUs; inference wants a fraction
11
scheduler
kube-scheduler / Kueue / Volcano / Slurm · places the pod
no gang scheduling → partial placement deadlock
10
GPU Operator
manages every layer above the kernel as DaemonSets
reconcile loop stuck; driver container crash-loops
9
NFD + GFD
labels the node: GPU model, memory, MIG profile, driver
wrong labels → scheduler mis-places the pod
8
device plugin
advertises nvidia.com/gpu to the kubelet
crashes → nvidia.com/gpu: 0, pods stuck Pending
7
containerd / runc
runs the OCI container on the node
default_runtime not set to nvidia → no GPU in pod
6
Container Toolkit
injects /dev/nvidia* + driver libs into the container
nvidia-smi works on host, fails in the pod
5
CUDA runtime
libcudart, cuDNN, NCCL · usually bundled in the wheel
driver ↔ toolkit ↔ cuDNN ↔ framework version skew
4
libcuda (driver API)
userspace entry to the driver; NVML backs nvidia-smi
libcuda older than the app's CUDA runtime needs
3
kernel driver · nvidia.ko
kernel-mode GPU access; exposes /dev/nvidia*
kernel-header mismatch → module won't load
2
firmware · VBIOS + GSP
on-GPU RISC-V core offloads driver work
GSP RPC timeout (Xid 119/120) → GPU hangs
1
silicon · GPU + NVSwitch + NIC
compute, HBM, and the fabric between cards
ECC, thermal throttle, fell off the bus (Xid 79)
