On-call triage
A GPU is misbehaving · what the Xid tells you
The split that matters: app fault restarts a pod, hardware fault cordons a node.
grep dmesg → NVRM: Xid (PCI:0000:xx:00): <code>
a nonzero Xid is never nothing; the code decides the next move
▽
app fault
restart the workload, look at the model, not the card
Xid 13graphics engine exception · usually illegal memory access
Xid 31MMU / page fault from application code
Xid 43GPU stopped processing, app-induced
hardware fault
cordon the node, reset or RMA the card
Xid 48 / 94contained ECC · drain politely, reset the GPU
Xid 95uncontained ECC · reset before anything restarts
Xid 63 / 64row remapping · watch the trend, RMA if it fails
Xid 79fell off the bus · reboot node, RMA if it recurs
Xid 119/120GSP firmware hung · reset the GPU
